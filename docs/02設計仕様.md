# zip2Addr 設計仕様書

## 1. 概要
本アプリケーションは、ブラウザ上で動作するHTML、CSS、JavaScriptを使用したWebツールです。郵便番号を入力し、ボタンを押すと外部の郵便番号検索APIを利用して住所を取得し、画面に表示します。

## 2. 技術スタック
- **構造 (HTML5)**: ページのレイアウトと要素の定義
- **見た目 (CSS3)**: 画面のデザイン、レスポンシブ対応
- **挙動 (JavaScript)**: APIとの通信、画面へのデータ表示

## 3. システム構成・API利用について
外部の公開APIを利用して住所データを取得します。
- **利用API**: [ZIPCloud](https://zipcloud.ibsnet.co.jp/)
  - 理由: 登録不要で利用可能、APIキーの管理が不要で、小規模なアプリケーション開発に適しているため。

### API利用方法
JavaScriptの `fetch` APIを使用してデータを取得します。

- **リクエストURL例**:
  `https://zipcloud.ibsnet.co.jp/api/search?zipcode=1000001`
- **処理の流れ**:
  1. ユーザーが入力した郵便番号をURLに付与してGETリクエストを送信。
  2. APIからのレスポンス（JSON形式）を受け取る。
  3. JSON内の `results` 情報を解析し、住所情報を抽出する。
  4. 住所が存在しない場合やエラー時は、適切なメッセージを表示する。

## 4. 画面・実装仕様
### 4.1. 画面構成
- **入力エリア**: 郵便番号入力欄 (input type="text")、検索ボタン (button)
- **表示エリア**: 住所表示用エリア (div)

### 4.2. 機能仕様
1. **入力チェック**: 郵便番号が半角数字7桁であることを入力時にバリデーションする。ハイフン（-）が含まれている場合は除去して処理する。
2. **検索実行**: ボタン押下時、または入力欄でEnterキー押下時、入力された値でAPIをコールする。
3. **データ表示**: APIからのレスポンスを元に、以下のように表示する。
   - 成功時: `住所（カナ）` を上に、その下に `都道府県 + 市区町村 + 町域` を表示。
   - 失敗時/エラー時: 住所表示エリア内に大きな赤文字でエラーメッセージを表示。

## 5. ファイル構成
- `index.html`: 全体の構造と要素
- `src/style.css`: 画面のデザイン
- `src/script.js`: 検索処理とAPI通信ロジック
